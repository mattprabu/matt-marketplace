# Notion Schema Reference

## Parent page setup

Before using Notion output, the user provides a **parent page URL or ID**. All
analyzed properties become child pages under this parent.

Ask the user on first Notion use in a session:

> Paste the URL of the Notion page where I should save property analyses.
> I will create one child page per property and a Portfolio Scorecard
> database inside it.

Store the parent ID in `/home/claude/.property-analyzer-config.json` for
the session only. Do not persist across conversations.

---

## Property page structure

Each property becomes a Notion page with:

**Title format**: `[Address] — [Emoji] [Verdict]`

Example: `4521 Oak Ridge Dr, Tampa, FL 33625 — 🟢 GREEN`

**Body blocks (in order)**:

1. Heading (H2) with price and spec line
2. Verdict callout block (green, yellow, or red background)
3. Composite score paragraph
4. "Key Metrics" heading with a toggle containing metric bullets
5. "Pros" heading with bullets (only metrics that scored 2)
6. "Cons" heading with bullets (only metrics that scored 0)
7. "Watch Items" heading with bullets (only metrics that scored 1)
8. "Risk Factors" heading with orange callouts (one per risk)
9. "Cash Flow Summary" heading with two summary paragraphs
10. Toggle "Comparable sales" (collapsed by default)
11. Toggle "Financial assumptions used" (collapsed)
12. "Sources" heading with listing URL bullet
13. Divider
14. Italic paragraph — disclaimer

Generated by `scripts/notion_export.py` function `build_property_page_blocks()`.

## Portfolio Scorecard database

A Notion database named `Portfolio Scorecard` lives inside the parent page.
Every analysis adds one row. Use this to rank, filter, and compare the full
pipeline of properties considered.

### Columns

| Name | Type | Notes |
|---|---|---|
| Property | Title | Full address |
| Verdict | Select | Options: Green, Yellow, Red (with color tags) |
| Price | Number (dollar) | Asking price at analysis time |
| Monthly Cash Flow | Number (dollar) | Net monthly after all expenses + debt |
| Cap Rate | Number (percent) | Stored as a number, formatted as percent |
| CoC | Number (percent) | Cash-on-cash return |
| DSCR | Number | 2 decimals |
| Analysis Date | Date | ISO date (YYYY-MM-DD) |
| Link to Page | URL | Back-link to the full property analysis page |

### Creating the database

On first Notion export in a workspace that does not have the scorecard yet,
create the database using the `Notion:notion-create-database` tool. The
schema to use is returned by `scripts/notion_export.py` function
`portfolio_db_schema()`.

Check for existence first via `notion-search` or `notion-fetch` on the
parent page's children. Do not create duplicates.

### Useful views to suggest the user creates manually

- **By Verdict**: board view grouped on Verdict
- **Top Cash Flow**: table sorted by Monthly Cash Flow descending, filtered
  to Verdict not equal to Red
- **Recent analyses**: sorted by Analysis Date descending, limit 10

## Comparison mode output

When comparison is run (2 or more properties), in addition to the individual
pages, create a standalone comparison page under the parent:

**Title format**: `Comparison — [N] properties — [date]`

Example: `Comparison — 3 properties — 2026-04-20`

**Body blocks**:

1. "Properties Compared" heading
2. Bulleted list, one bullet per property linking to its analysis page
3. "Side-by-side" heading
4. A Notion table block (1 header row + 1 row per metric)
5. "Winner by Metric" heading
6. Paragraph naming the winning property for cash flow, cap rate, CoC, IRR
7. "Recommendation" heading
8. Paragraph with final recommendation (or "none of these — keep looking")
